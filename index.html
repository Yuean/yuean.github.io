字符串类题目

字符串类题目

摘自LeetCode和《剑指Offer》还有《程序员面试指南》之类的。

翻转字符串？ 翻转数组？

reverse(0,i-1);
reverse(i, n-1);
reverse(0,n-1);

替换空格？合拼数组

如果只能使用字符串本身的空间，那么先遍历一次数组计算出空格的个数，并可以计算出总长度，每替换一个空格，长度增加2，如果保证字符串空间足够大。两个指针，一个指向字符串末尾，一个指向替换后的末尾，逐个覆盖

打印从1到最大的n位数

解法一：不停加1，第一位产生进位的时候表示溢出。输出的时候需要把前面的0过滤掉。

解法二：全排列！递归！当length增长到n-1的时候才输出

字符串的排列

把字符串看成是两部分，第一部分是第一个字符，第二部分是后面的所有字符。拿第一格和后面的字符逐个交换。如果到’\0’，就将字符串输出

字符串的组合

分成两部分，求n-1个字符中长度m-1的组合和n-1个字符中长度为m的组合

第一次只出现一次的字符

用哈希来做就好了。256就够了，数组的大小。

从第一个字符串中删除第二个字符串中出现过的所有字符

用数组实现的简单哈希表存储第二个字符串。

删除字符串中所有重复出现的字符。

同上

变位词

用哈希。扫描第一个字符串，所在值+1.扫描第二个字符串，所在值-1.如果哈希中所有值都为0，则是变位词。

把字符串转换为整数（atoi）

注意空串，非法的输入，用全局变量来标识是否非法。先判断是不是+-号，如果后面就没了，那么非法。在判断是不是在’0’~’9’，numstr-‘0’得到数字，还要考虑溢出的情况。使用辅助变量保存+-号，(!minus && num>0x7FFFFFFF) || (minus && (signed int)0x800000000)

正则表达式匹配

‘.’表示任意字符，’*’表示出现任意次，包括0次

根据状态机，选择后移两位（忽略*）或者保持不变。这两种情况相或。

表示数值的字符串

判断字符串是否表示数值。
表示数值的字符串遵循如下模式：
[sign]integral-digits[.[fractional-digits]][e|E[sign]exponential-digits]
isNumeric() scanDigit() isExponential()

字符串中第一个不重复的字符

哈希表

数组类题目

数组类题目

摘自LeetCode和《剑指Offer》还有《程序员面试指南》之类的。

二维数组中的查找：

前提：有序。

以左下角或者右上角为基准查找

数组中只出现一次的数字

用异或

数组中只出现一次的两个数字

先异或一次，然后根据异或的结果，以第一个为1的位置来将原数组划分

数组中出现超过一半的数字

解法一:partition算法 判断是不是数组中间

解法二:相同+1 不同-1 记录次数不为0的数

注意：partition解法需要修改数组！

数组中找出任意一个重复的数字，数字范围在0~n-1之间

重排数组，从第一个数字开始，检查这个数字是否等于i，如果是就扫描下一个数字，如果不是，将其和第m个数字相比较，如果相等，找到重复的，如果不相等，与第m位交换。

最小的k个数

解法一 同上
解法二 用容器 最小堆 set multiset

上台阶，一次可以上1级或者2级

动态规划
f(n) = f(n-1)+f(n-2)

子数组的最大和

动态规划法

fi = pData[i] i=0, fi-1<=0

fi = fi-1 + pData[i] fi-1>0

旋转数组中的最小数字

二分法，双指针。第一个指针指向头，第二个指针指向末尾。如果两个指针的值和中间的值相同，则用顺序查找。

斐波那契矩阵快速幂求

a^n = a^(n/2)·a^(n/2) 偶数

a^(n-1/2)·a^(n-1/2)·a 奇数

数值的整数次方

也可以使用上述公式，使用递归。

调整数组使奇数位于偶数前，负数放在非负数前，被整除和不被整除

双指针法。p1遇到偶数停，p2遇到奇数则交换。考虑可扩展性，利用函数 参数为bool(*func)(int)

顺时针打印矩阵

注意判断条件。从左到右打印一行，从上到下。从右到左，从下到上。
结束的条件column>startX2 && row>startY2

###　输入含8个数字的数组，是否能把8个数字分别放在正方体的8个定点上，是的正方体上三组相对的面上4个顶点的和相等
排列，然后判断a1+a2+a3+a4==a5+a6+a7+a8,a1+a3+a5+a7==a2+a4+a6+a8,a1+a2+a5+a6==a3+a4+a7+a8

从1到n中1出现的次数

直观解法：求余判断是否为1

把数组排成最小的数

改变比较函数的定义，如果nm<mn，则n<m。注意是大数问题，转换成字符串来做。用qsort来排序。

丑数

丑数是另一个丑数乘以2,3,5的结果，用数组来保存。假设已有的最大丑数为M，先将前面每一个乘以2。将第一个大于M的结果记为M2，同理到M3和M5.当前最大丑数T2T3T5每次取3个数中最小的一个，index+1，知道index到达目标值。

数组中的逆序对

分治合拼。合拼的时候统计逆序对的数目。先统计内部的逆序对数，再统计相邻子数组的数目。统计完要把子数组排序。两个指针指向两个子数组的末尾。如果第一个子数组中的数字大于第二个子数组中的数字，逆序对的数目等于第二个子数组中剩余数字的个数。。比较完后，把较大的数字从后往前复制到辅助数组中。

数字在排序数组中出现的次数

GetFirstK，GetLastK，都用二分法

和为S的两个数字

双指针法，p1指向第一个，p2指向最后一个。和大于目标，p2前移1位，小于，则p1后移1位

和为S的连续整数序列

small初始化为1，big初始化为2。增加big，直到大于等于目标，等于则的话，增加big，大于的话，增加small

构建乘积数组，不能用除法

给定A[0,1,….,n-1]，构建B，B[i]=A[0]A[1]…A[n-1]

如果可以用除法，用公式求A1…AN再除以Aj

把B[i]看成是A[0]…A[i-1]和A[i+1]…A[n-1]两部分的乘积，使用矩阵来计算

C[i] = A[0]…A[i-1] D[i] = A[i+1]…A[n-1]
C可以自上而下地求，D可以自下而上来求
时间复杂度O(n)

链表类题目

链表类题目

这里整理了一些与链表有关的题目，主要来自LeetCode和《剑指Offer》还有《程序员面试指南》之类的。

判断是否有环？

快慢指针法！一个走一步，一个走两步。

找到中间节点？

同上

判断是否回文？

同上。先根据节点个数的奇偶来判断，然后将后半段链表翻转。前半部分后后半部分同时遍历，如果都相同，就是回文啦！

链表中环的入口节点？

如果环的节点个数已知为n。可以用双指针法，p1先移动n步，然后p1p2一起移动，直到相遇。

如何获得环的节点个数呢？快慢指针法一定会在环中相遇。这时开始计数，如果走到原来的节点，就可以知道环的长度了

翻转链表？

pReverseHead，pNode，pNext，pPrev

Reverse a linked list from position m to n. Do it in-place and in one-pass.

使用pPrev指向m-1的节点就可以了

判断是否有交点？

各遍历一次获取长度，长的先走

从尾到头输出链表？

如果可以允许改变链表的结构，直接将其翻转再遍历。不可以改变的话，就用栈来实现。因为栈本身是递归，所以可以用递归来实现！不过递归可能会栈溢出！

在O(1)时间删除链表结点？

正常思路，遍历，如果下一个节点是要删除的节点，就将当前节点的next指向删除节点的后一个节点，复杂度为On

将下一个节点的内容复制到当前节点，然后把下一个节点删除掉！如果节点在尾部，则从头开始找到其前一个节点。如果是头节点，删除后要设为NULL

注意因为是O1，无法判断是要删除的节点是否在链表中

合拼两个排序的链表？

递归。pMergedHead=pHead1/pHead2 pMergedHead=m_pNext = Merge(pHead1/2->m_pNext, pHead1/2)

查找/删除链表中倒数第k个节点？

直观解法：遍历链表两次。第一次获取长度。第二次遍历n-k+1次

遍历一次解法：双指针法。第一个指针向前走k-1步。第二个指针保持不动。从第k步开始，第二个指针从头开始遍历。当第一个指针到达尾节点时，第二个指针刚好在第k个节点上。但是要注意是否为空指针，节点总数是否小于k，k为0的时候，k-1是什么？

复杂链表的复制？

第一种方法，分两步。第一步复制所有节点，第二步找Sibling

第二种方法，用哈希表将N和N’配对，根据N来找Sibling

第三种方法，对于每一个节点N，复制一个节点N’在其后，形成新链，如果N的Sibling是S，则N’的Sibling是S的下一个节点S’，最后拆分为两条链，一条是奇数节点，即原链表，第二条是偶数节点的，是复制后的链表

删除重复的节点？

如果是有序链表，使用pPrev指向前一个节点。

排序链表、利用插入排序排序链表

如果是单纯要排序链表，用合拼排序就对了！

删除所有重复节点，包括自身

借助变量needtobedelete来保存要删除的值然后遍历

Sort a linked list in O(n log n) time using constant space complexity.

使用合拼排序。

链表中的两两交换？

借助pPrev

Given a linked list and a value x, partition it such that all nodes less than x come before nodes greater than or equal to x.

这题啊，注意看题意啊，别人没说要排序啊，遍历的时候，如果有大于x的，就标记一下，这意味着后面出现的小于x的值都得插入在这个值后面

Given a singly linked list where elements are sorted in ascending order, convert it to a height balanced BST.

BST的中序遍历就是排好序的链表了。所以问题变为如何根据中序遍历建树。

Given a singly linked list L: L0→L1→…→Ln-1→Ln, reorder it to: L0→Ln→L1→Ln-1→L2→Ln-2→…You must do this in-place without altering the nodes’ values.

用stack就搞定了！

Given a list, rotate the list to the right by k places, where k is non-negative.

旋转的题目，注意旋转的数字可能比本身的长度大！旋转的位数等于本身的长度的话，相当于不动。所以长度大也不怕，模一下长度就好了！

Merge k sorted linked lists and return it as one sorted list. Analyze and describe its complexity.

可以沿用原先的合拼两个有序链表的方法，使用递归和二分的方法。

还可以用堆排序。但每次都要调整堆太不科学了。

用优先队列就没问题了！先把各链表的头压入优先队列中，然后取出最小的一个，访问其下一个节点，如果有，就压入优先队列中。
