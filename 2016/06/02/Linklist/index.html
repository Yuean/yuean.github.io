LeetCode 链表类题目

No.2 Add Two Numbers

这题目一开始看错题意了。实际上就是两个数相加，这两个数是用链表表示的，比如342就是2->4->3。所以只需要在遍历的时候将两数相加，记录的进位插在链表后面，直到两条链表都遍历完毕就好了。

No.19 Remove Nth Node From End of List

直观的解法：遍历一次链表获取链表的长度N，然后从头开始遍历到N-n的位置，然后删掉那个结点。

只遍历一次的解法：遍历的时候用堆栈保存节点，遍历完后在堆栈直接获取目标节点进行操作。或者直接用递归。

No.21 Merge Two Sorted Lists

反正两条链分别遍历，哪个节点大一些就排在前边作为父节点。《剑指offer》也有提到，用的是递归。

No.23 Merge k Sorted Lists

可以沿用原先的合拼两个有序链表的方法，使用递归和二分的方法。

还可以用堆排序。但每次都要调整堆太不科学了。

用优先队列就没问题了！先把各链表的头压入优先队列中，然后取出最小的一个，访问其下一个节点，如果有，就压入优先队列中。

No.24 Swap Nodes in Pairs

借助指向前一个节点的指针pPrev

No.25 Reverse Nodes in k-Group

这题虽然是Hard，其实也不难的。先遍历一次获取链表的长度，这样就知道需要翻转多少次了。至于翻转用之前的方法就可以了。

No.61 Rotate List

先遍历一次链表获取链表的长度，根据输入的k，模一下链表的长度得到新的链表头的位置，遍历到新链表头的位置，将其设为新的链表头，将旧链表头接在其末端。

No.82 Remove Duplicates from Sorted List II

借助一个变量needdelnum保存当前节点的值，然后遍历后一个节点是否相同，然相同则删除。注意删除头节点的情况。

由于题目说明了给定的是一个排好序的链表，所以重复的节点肯定是连续在一起的，就不需要考虑隔着的情况了。

No.83 Remove Duplicates from Sorted List

这题跟上一题一样，只是不要求删除自身，也就是多个重复的节点只保留一个。

No.86 Partition List

这题做的时候也是没看题意啊，以为要排序，其实别人没说要排序，遍历的时候，如果有大于x的，就标记一下，这意味着后面出现的小于x的值都得插入在这个值后面。

No.92 Reverse Linked List II

使用pPrev指向m-1的节点，然后将n-m作为步数，用头插入的方法翻转中间的部分，接在pPrev后面。

No.109 Convert Sorted List to Binary Search Tree

BST的中序遍历就是排好序的链表了。所以问题变为如何根据中序遍历建树。关于树的遍历会单独post一篇来讲。

No.138 Copy List with Random Pointer

这种题目《剑指offer》有提到，解法如下：

第一种方法，分两步。第一步复制所有节点，第二步复制另一个指针的指向

第二种方法，用哈希表将节点N和节点N的指针指向的节点N’配对，根据N来找指针的指向

第三种方法，对于每一个节点N，复制一个节点N’在其后，形成新链，如果N的另一个指针指向的是S，则N’的另一个指针指向S的下一个节点S’，最后拆分为两条链，一条是奇数节点，即原链表，第二条是偶数节点的，是复制后的链表

No.141 Linked List Cycle

判断是否有环，用快慢指针法。一个指针每次走一步，另一个指针每次走两步，如果有环，则这两个指针必定会相遇。

No.142 Linked List Cycle II

获取环的入口节点。这题《剑指offer》也有提到， 如果环的节点个数已知为n。可以用双指针法，p1先移动n步，然后p1p2一起移动，直到相遇。

那么，如何获得环的节点个数呢？

由于快慢指针法一定会在环中相遇。这时开始计数，如果走到原来的节点，就可以知道环的长度了

No.147 Insertion Sort List

用插入排序算法排序链表。知道插入排序就行了吧。

No.148 Sort List

这题要求时间复杂度是$O(n\log{n})$且空间复杂度为常数。使用合拼排序即可。

No.160 Intersection of Two Linked Lists

找到两个链表的交点。对两个链表各遍历一次获取其长度M和N，长的先走$|M-N|$步，然后一起走。

No.203 Remove Linked List Elements

删除指定值的节点。用指针记录前一个的节点就好了。

No.206 Reverse Linked List

翻转链表。可以用递归或者迭代来完成。我倾向用迭代来做的，这样不会爆栈。

No.234 Palindrome Linked List

判断是否是回文链表。先用快慢指针法走到链表中间。然后据节点个数的奇偶来判断，然后将后半段链表翻转。前半部分和后半部分同时遍历，如果都相同，那就是回文啦！

No.237 Delete Node in a Linked List

删除链表中的一个节点，有个特殊条件，只給你要删的那个节点，不给你整条链，也就是说你不能访问前一个节点！怎么办呢？只要复制后面的节点，然后把后面的节点删掉就可以了！
